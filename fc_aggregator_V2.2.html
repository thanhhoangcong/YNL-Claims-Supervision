🏭 HRBP & FC Role Matrix Aggregator

HRBP (Human Resources Business Partner) is the strategic HR function embedded within YNL Garments' operational structure, serving as the primary link between business units and the central People & Organisation division. HRBPs work directly alongside department heads and line managers to ensure that every team is staffed with the right people, equipped with the right competencies, and developed along a structured career pathway aligned with the company's long-term objectives.

As part of the FC (Functional Competency) Framework, the HRBP team maintains a comprehensive Role Matrix for every department across the YNL Garments organisation. Each Role Matrix defines the competencies required for every position, the proficiency levels expected at each grade, and — when current data is available — the actual assessed level of each employee. This structured documentation enables HR and department leaders to identify skill gaps, plan targeted learning interventions, prioritise succession pipelines, and make objective, evidence-based decisions on promotions and role assignments.

📊 About This Application

The FC Role Matrix Aggregator is an offline, browser-based tool developed to consolidate individual department Role Matrix HTML reports into a single, unified, and dynamic synthetic dashboard. Prior to this tool, HR Business Partners were required to open and cross-reference each department's report file separately — a time-consuming process prone to inconsistency. This application eliminates that overhead by allowing users to load an entire folder of reports at once and immediately access an aggregated, organisation-wide view.

Once reports are loaded, the Aggregator provides the following capabilities:

  • A Department Summary table showing competency counts, roles mapped, current-data coverage rates, average competency gap scores, and report dates — all in one scrollable view
  • A Total Roles roster listing every position across all loaded departments, with FC codes, employee names, and ID numbers, filterable and searchable in real time
  • A Competency Gap Analysis that ranks all FC competencies by their average gap score (Current − Required) to highlight where the organisation is strongest and where targeted development is most urgent
  • A Name / ID Search function enabling HR to instantly locate any individual across all departments without navigating department by department
  • Integrated links to the FC Database (Full Role Matrix, Competency Library, and Competency Dictionary) for deep-dive reference
  • One-click Print PDF and CSV Export for offline reporting and data sharing with stakeholders

All data is processed entirely within your local browser. No files are uploaded to any server and no internet connection is required. Reports can be saved as a self-contained HTML snapshot for archiving or sharing.

1. Overview

The FC Role Matrix Aggregator is an internal HR tool for YNL Garments that consolidates individual department Role Matrix HTML reports into a single unified view. It lets you compare competency requirements and current levels across all departments, search by employee name or ID, export data, and produce printable reports — all without any server or internet connection.

2. Loading Department Reports

Expected filename format:

Role_Matrix_YNL_DepartmentName_YYYY-MM-DD.html

Files that do not match this pattern are skipped with a warning toast.

Three ways to load files:

A
📁 Load Folder — Click this button and select the entire folder containing the report files. All matching .html files in that folder are loaded at once.
B
Drag & Drop — Drag one or more report HTML files directly onto the drop zone shown on the empty-state screen.
C
Click the drop zone — Click the dashed box on the empty-state screen to open a standard file picker and choose individual files.
Re-loading a report that is already loaded automatically updates it in place (detected by filename or department name). Any name edits you made are preserved.
3. Stat Cards (Navigation Bar)

Four clickable cards appear below the header once reports are loaded:

  • Departments — total number of loaded departments. Click to open a sortable list.
  • Total Roles — sum of all roles across all departments. Click to list every role with name and ID.
  • Unique Competencies — number of distinct competency names. Click to see which departments use each and the average required level.
  • Roles w/ Current Data — roles that have a current proficiency score recorded. Click to list them with average gap indicators.

All four modals update in real time as more reports are loaded.

4. Overview Table

The main area shows a summary table with one row per department:

  • Department — click any row to open that department's full detail modal.
  • Roles — number of roles defined in that report.
  • Competencies — number of competency columns in that report.
  • Coverage — percentage of roles that have current data entered.
  • Avg Gap — average difference between current and required proficiency for roles with data. Green = meeting or exceeding requirements; red = below requirements.
  • Date — report date extracted from the filename.
  • Edit button — rename the department label (see below).

Editing a department name:

1
Click the Edit button on any row in the overview table.
2
Type the new name in the prompt dialog and click OK.
3
The name updates everywhere (overview, sidebar, modals). It is preserved when you use 💾 Save or 💾 Save Report.
5. Search (Name / ID No.)

The 🔍 Name / ID No. search box in the header searches all loaded departments simultaneously.

  • Type any part of an employee name, ID number, or role title.
  • Matching text is highlighted in yellow.
  • Click any result row to open that department's full detail modal.
  • Press Esc or clear the box to return to the Overview table.
6. Department Detail Modal

Click any department row (or sidebar item) to open a detailed modal showing:

  • Role Matrix table — every role with columns for each competency. For each competency: Req (required level), Cur (current level), Gap (difference). Green ✔ or + means meeting/exceeding; red negative means a shortfall.
  • Level Key — colour-coded circles 1–6 indicating proficiency levels.
  • Competency Gap Charts — one bar chart per role showing required vs. current levels and any gap visually.

Modal actions:

  • 🖨️ Print PDF — opens a clean print-optimised version of the modal in a new window and triggers the print dialog.
  • 📊 CSV — downloads a CSV file for this department containing all roles and competency data.
  • ✕ — close the modal (or click outside it).
7. Database Viewer Buttons

Three buttons open the FC Framework Database in a separate window:

  • 🎯 Role Matrix — opens the role matrix reference tab.
  • 📚 Competencies — opens the competency definitions tab.
  • 📖 Dictionary — opens the competency dictionary tab.

In the working version, database/database.html is automatically loaded in the background when the tool opens — no action required. In a saved version (generated by 💾 Save), the database content is decoded from the embedded copy — no external file is needed at all.

8. Sidebar — Loaded Reports

The panel on the right lists all loaded department reports.

  • Filter box — type to filter the sidebar list by department name. Press Esc to clear.
  • Click a card — opens that department's detail modal.
  • ✕ (remove) button — top-right of each card — removes that department from the session without affecting the original file.
  • The count badge next to "Loaded Reports" shows how many are currently loaded.
9. 💾 Save — Working Copy with Embedded Data

Creates a self-contained copy of the tool with all currently loaded report data and the FC Database baked in. This is the file you use day-to-day.

1
Click 💾 Save. No file picker appears — the tool automatically reads database/database.html (already loaded in the background) and embeds it without any extra step.
2
A file is downloaded with the same filename as the working tool (e.g. fc_aggregator_V2.2_blank.html). Replace your working copy with this downloaded file.
All buttons remain active in this saved version — you can continue loading new reports, editing names, and saving again. The 🎯/📚/📖 Database buttons work from the embedded copy, so no external files are required.
10. 💾 Save Report — Read-Only Snapshot

Creates a dated snapshot of the current state for sharing or archiving. The file name includes today's date and time (e.g. HRBP_FC_Aggregated_Report_V2.2_2026-04-09_Time_0830.html).

  • All report data is embedded — recipients can open and browse without loading anything.
  • The database is carried over automatically from the background-loaded copy.
  • The 📁 Load Folder, 💾 Save, and 💾 Save Report buttons are hidden in snapshot files — the file is intentionally view-only.

Manifest auto-update: Immediately after the snapshot is downloaded, the tool also downloads an updated manifest.json file that lists all saved report filenames (newest first). Upload both files to the updated_status/ folder on the server so the report index stays current.

Workflow tip: Use 💾 Save to update your working copy (with the latest report data), then use 💾 Save Report to produce a fully-embedded, shareable snapshot — and upload both the HTML and manifest.json to updated_status/.
11. 🖨️ Print PDF & 📊 CSV

🖨️ Print PDF — prints the current main view (overview table or search results) as a landscape A4 PDF via the browser's print dialog. To print a single department's matrix, use the Print button inside its detail modal instead.

📊 CSV — exports a summary CSV of all roles across all departments (Department, FC Code, Role, Name, ID No., Date). For a competency-level CSV of one department, use the CSV button inside that department's modal.

12. 🗑️ Clear

Removes all loaded reports and resets the tool to the empty state. The page does not refresh — your working file is unchanged. Use this when you want to start fresh with a different set of departments.

13. Tips & Notes
  • Database auto-load — when running as the live working file, database/database.html is fetched silently in the background as soon as the page opens. The 🎯/📚/📖 buttons and 💾 Save use this automatically — no manual selection needed.
  • File:// limitations — the tool runs entirely in your browser from the local file system. Auto-loading the database and some other features (logo embedding) require running from a local web server (e.g. Firebase Hosting or VS Code Live Server) rather than opening directly via file://. If you open the file directly, use a browser launched with --allow-file-access-from-files, or use a saved/embedded version.
  • No internet required — Chart.js is loaded once from a CDN on first open; after that, all features work offline.
  • Clock — the live clock in the header shows the current time and date. It is decorative and has no effect on saving or exporting.
  • Department order — the overview table sorts alphabetically. Sidebar order reflects loading order.
  • Duplicate detection — if you load a file with the same department name or filename as one already loaded, it is updated in place. Custom names you edited are preserved.
  • Browser compatibility — tested in Google Chrome. Most features work in Edge and Firefox; the file-picker and auto-load flows are optimised for Chrome.
